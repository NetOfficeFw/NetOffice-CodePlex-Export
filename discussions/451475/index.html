you dont need anything to do. delete the old toolbox and start the DeveloperToolbox.exe (xcopy deployment)

update: its a bug in the settings file handler.
i do an update today for the toolbox.

hotfix:
got to the directory Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData)
(it's C:\AppData in Win7, not sure...) and replace the content of the file DeveloperToolbox.Settings.xml

with:
<?xml version="1.0" encoding="utf-8"?>
<Application StartMinimized="False" MinimizeToTray="False" StartWithWindows="False" Language="0" Version="1.2.0.0">
  <Controls>
    <ApplicationObserver>
      <Control>
        <Excel Enabled="True" />
        <Winword Enabled="True" />
        <Outlook Enabled="False" />
        <PowerPnt Enabled="True" />
        <MsAccess Enabled="False" />
        <Project Enabled="False" />
        <Visio Enabled="False" />
        <HotKey Enabled="False" Keys="131153" />
        <Tray Enabled="False" />
      </Control>
    </ApplicationObserver>
    <RegistryEditor>
      <AskBeforeDelete>false</AskBeforeDelete>
    </RegistryEditor>
    <AddinGuard>
      <Active>false</Active>
      <SetLoadBehavior>false</SetLoadBehavior>
      <TrayNotify>false</TrayNotify>
    </AddinGuard>
    <OfficeCompatibility>
    </OfficeCompatibility>
    <OutlookSecurity>
      <SupressEnabled>false</SupressEnabled>
    </OutlookSecurity>
  </Controls>
</Application>
hope this helps
