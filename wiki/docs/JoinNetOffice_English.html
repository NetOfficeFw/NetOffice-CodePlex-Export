1.) You want improve the NetOffice example code? 

NetOffice has a lot simple example projects. You want create more examples? Welcome!

2.) You want improve the NetOffice Visual Studio integration?

NetOffice offers a project wizard for Visual Studio. This is an important feature for the future of NetOffice. NetOffice needs more features like ribbon designer and more for the future. Do you have good ideas here? NetOffice needs you!


3.) You may want improve the NetOffice documentation?

NetOffice has a documentation of course, also f.a.q. technical reference and so on but its difficult to create one documentation for different kind of users. You have ideas to make the documentation better as before? Lets talk about!

 

4.) You want extend the NetOffice Developer Toolbox?

The NetOffice Developer Toolbox is an important extension for NetOffice specially the Office Compatibility component. More components with helpfull functionality is nice. You are an office developer ans something in you daily working sucks?  You have an idea for a tool to make your situation better? Intgegrate your idea in the Developer Toolbox please! 

 

5.) You are an expert for a spezific Office application?

NetOffice want to be a good choice for small and big projects. Independent from the .Net Framework version or the programming language. Any Office application is her own universe. Do you a master of the universe? Do you create solution in for an Office application on a professional level? In this case you have ideas, feedback and consulting services for NetOffice. This helps to make sure the NetOffice development goes in the right direction.


6.) You want keep the quality of NetOffice?

The development of Office solutions is like a dance with the devil(van Halen). The testing of new NetOffice releases may need more time as you think. Limitless virtuell machines for different office versions and system are in use. But anyway, any system is her own world, specially trouble systems with different office versions is very challenging. NetOffice want to be an API for the real world. You want to help NetOffice here and execute the test programs on your system, give feedback and anlyzing problems(if occurs). This is a valuable service for NetOffice.


7.) You are an expert in Linq?

NetOffice supports Linq. All collections implements IEnumerable<T>. But queries in Ling2Objects are not very efficient in relationship with COM Interop. This can be better: Example  in C# and More info
No worries, its not the idea you have to create Linq extensions for all the types in NetOffice, on type is enough. the technology from the NetOffice Codegenerator makes it possible to apply one single extension module (for example Excel.Range or Outlook.Contact) to all the other Office types.
