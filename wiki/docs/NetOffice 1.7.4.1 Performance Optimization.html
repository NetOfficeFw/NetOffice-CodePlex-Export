NetOffice 1.7.4.1 Performance Optimizations

 

- avoid analyzing well known proxies which is not a base type for someone,
  specialy event arguments

- improve building the latebinding argument array

- improve sink helper performance in event interfaces by caching reflection operations

- remove duplicated proxy information calls in Core(GetParentLibraryID, etc.)

- implement COMProxyShare to avoid calling the remote server when its possible

- spend a fixed type cache for well known duplicate types instead of try search the whole appdomain

- improve initialization process
